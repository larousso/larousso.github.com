## Fonctions pures, fonctions totales, transparence référentielle

En programmation fonctionnelle, on parle de manipulation de fonctions (pas con, c'est dans le nom en même temps !). Les fonctions peuvent avoir plusieurs propriétés, elles peuvent être
* Pures
* Totales
* Transparentes référentiellement (pas sûr que ça se dise ...)

### Fonctions pures

Pour qu'une fonction soit pure, elle doit retourner le même résultat avec les mêmes paramètres d'entrée.

Par exemple, cette fonction est pure :
```java
int additionner(int a, int b) {
    return a + b;
}
```

Si on exécute plusieurs fois `additionner(2, 3)` on obtiendra toujours `5`.

Utiliser des fonctions pures est très confortable, il n'y a pas de surprise, c'est parfait !

### Transparence référentielle

La transparence référentielle, c'est lorsque substituer une fonction par une variable et manipuler cette variable revient à manipuler directement la fonction.
